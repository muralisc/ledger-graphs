# Test suite context — ledger-graphs/v1

## What this tests

Five gnuplot/ledger shell scripts in `../v1/` that generate financial graphs:

| Script | What it produces |
|--------|-----------------|
| `1-yearly-last10year-cumulative_asset_expense.sh` | Asset/expense cumulative growth + FI projections |
| `2-monthly-lastmonth-income_expense.sh` | Monthly income vs expense bar chart (14-month window) |
| `3-monthly-lastmonth-expense_accounts.sh` | Monthly expense sub-account breakdown (13-month window) |
| `4-daily-lastmonth-expense.sh` | Daily expense time series (all history) |
| `5-monthly-last1year-income_expense.sh` | Rolling 12-month income/expense |

All scripts take `LEDGER_FILE`, optional `LEDGER_PRICE_DB`, and `FOLDER` (output dir)
as positional arguments. `FOLDER` was moved outside the scripts specifically to enable
testing — previously it was hardcoded inside each script.

## How to run

```bash
# Run tests (compare against golden files)
bash tests/run_tests.sh

# Regenerate golden files after intentional changes
bash tests/create_golden.sh
```

`run_tests.sh` exits non-zero on any failure. 74 assertions total across 5 scripts.

## Key design decisions

### LEDGER_TEST_DATE
All scripts and `lib.sh` use `${LEDGER_TEST_DATE:-$(date +"%Y-%m-%d")}` instead of
calling `date` directly. Setting this env var pins "today" across the entire run,
making all output files fully deterministic and safe for `diff`.

- **Fixed test value**: `2026-06-01` (first of month, within test data window)
- **Affects**: `net_yearly` in lib.sh, `CURRENT_MONTH_START`/`START_TIME` in scripts
  2/3/5, graph titles in all scripts, projection start date in script 1.

### Golden file testing
Each script's intermediate data files (`.tmp`, `.txt`) are diffed against committed
golden files in `tests/golden/script{1..5}/`. PNGs are only checked for existence
(non-empty). Rationale: data files encode the correctness of ledger queries and data
transformations; PNGs are gnuplot rendering output and not worth byte-comparing.

When to regenerate golden files (`bash tests/create_golden.sh`):
- `test.ledger` or `test.pricedb` changed
- A script's ledger query or data transformation intentionally changed
- `LEDGER_TEST_DATE` changed in `run_tests.sh`

### Test data (`test.ledger`)
Generated by `generate_test_data.sh` — **do not edit `test.ledger` by hand**.
Characteristics:
- 10 years of monthly GBP transactions (salary + 8 expense categories)
- Multi-currency: GBP (primary), INR (SBI savings interest), USD (Schwab interest),
  VUAG (Fidelity ISA ETF, quarterly purchase), GOOG (Schwab stock, biannual)
- Daily transactions for last 3 months (needed for script 4's daily graph)
- Special months for edge-case coverage:
  - 18mo ago: bonus salary (6500 GBP instead of 3000)
  - 12mo ago: high health expense (520 GBP — dental)
  - 9mo ago: big possession purchase (850 GBP — laptop)
  - 4mo ago: low-expense month (holiday pattern)
- All accounts declared → passes `ledger --strict`
- Balances to zero

Exchange rates fixed in `test.pricedb` (simplified round numbers):
- `INR 0.01 GBP`, `USD 0.80 GBP`

### Script 1 parameterisation
Script 1 hardcoded personal career milestone dates (Cisco→Meta job change) and a
target FI annual expense figure. These are now env vars with defaults:

| Env var | Default | Purpose |
|---------|---------|---------|
| `MILESTONE_DATE_NEW_JOB` | `2021-12` | Avg savings calculated at this date |
| `MILESTONE_DATE_OLD_JOB` | `2020-11` | Projection anchor + savings baseline |
| `YEARLY_EXPENSES_GBP` | `46000` | FI target calculation |
| `LEDGER_RUN_DATE` | falls back to `LEDGER_TEST_DATE` | Graph title date |

`run_tests.sh` overrides these to the same defaults (explicit rather than implicit)
so test intent is visible.

## Current gaps / future work

- **Script 1 projection files**: `graph1_meta_compound.tmp` and
  `graph1_cisco_compound.tmp` start from `LEDGER_TEST_DATE` — their values depend
  on the current asset balance which is computed from ALL history in the ledger file.
  With 10-year test data this is stable, but worth noting.

- **lib.sh unit tests**: `net_yearly`, `get_past12_mothly_avg_savings`, and
  `projection` are tested only indirectly via script 1. A targeted unit test with a
  tiny ledger (3-4 transactions with known exact values) would isolate lib.sh bugs.

- **Edge case: missing category months**: script 3 should gracefully handle a month
  where an expense category has zero transactions. Currently untested — add a month
  with no `Expenses:Posessions` to `test.ledger` and verify the plot still renders.

- **`ledger --strict` in scripts**: the scripts themselves don't pass `--strict` to
  ledger (only `ledger_b` in lib.sh doesn't either). Test data passes `--strict` but
  production data may have undeclared accounts that silently pass.

- **Parallel job failures**: scripts use `&` + `wait` for parallel ledger queries.
  If a background job fails, `wait` still exits 0. The test only checks the final
  exit code and file contents — a failed background query would produce an empty file
  caught by `check_exists`, but the error message would be lost to the log.

## File map

```
tests/
  CLAUDE.md                  ← this file
  generate_test_data.sh      ← source of truth for test.ledger; edit this, not test.ledger
  test.ledger                ← committed generated output (regenerate with generate_test_data.sh)
  test.pricedb               ← fixed INR/USD→GBP rates for deterministic conversion
  run_tests.sh               ← test runner; source of LEDGER_TEST_DATE and env overrides
  create_golden.sh           ← thin wrapper: calls run_tests.sh --update-golden
  golden/
    script1/                 ← 6 .tmp files
    script2/                 ← 3 .txt files + ledger_run_date.txt
    script3/                 ← 18 .txt files (8 categories × raw + moving avg, plus totals)
    script4/                 ← 1 .tmp file
    script5/                 ← 2 .tmp files + ledger_run_date.txt
```
